Taxonomy of LLM Architectures
Three Main Architectural Families:
1 Encoder-Only (BERT-style):
Bidirectional attention
Understanding-focused
Masked language modeling
2 Decoder-Only (GPT-style):
Unidirectional (causal) attention
Generation-focused
Autoregressive modeling
3 Encoder-Decoder (T5-style):
Separate encoding and decoding
Sequence-to-sequence tasks
Flexible architecture
Note:Most modern LLMs use decoder-only architecture for generative
tasks

GPT Architecture Details

Tied weights with input embeddings
Softmax for token probabilities

===== PAGE 9 =====
LLaMA Architecture Innovations
Key Technical Improvements:
1 Pre-normalization (RMSNorm):
Normalize inputs to each transformer sub-layer
Improves training stability
More efficient than LayerNorm
2 SwiGLU Activation:
SwiGLU(x) = Swish(xW)⊗(xV)
Replaces ReLU in feed-forward
Better performance than GELU
3 Rotary Position Embeddings (RoPE):
Relative position encoding
Better extrapolation to longer sequences
Encodes position through rotation
4 Grouped-Query Attention (LLaMA 2):
Multiple query heads share key/value heads
Reduces memory and compute
Maintains performance

Mixture of Experts (MoE)
Architecture Concept:
Multiple ”expert” sub-networks
Router network selects which experts to use
Only subset of experts active per token
Sparse activation for efficiency
Advantages:
1 Efficiency:Fewer active parameters per forward pass
2 Capacity:More total parameters without proportional compute
3 Specialization:Different experts for different patterns
4 Scalability:Easier to scale to massive sizes
Mathematical Formulation:

===== PAGE 22 =====
Attention Mechanism Variants
Evolution of Attention:
1 Multi-Head Attention (Original):
Separate Q, K, V for each head
Memory:O(n·h·d)
2 Multi-Query Attention (MQA):
Share single K, V across all heads
Memory:O(n·d)
Faster inference, slight performance drop
3 Grouped-Query Attention (GQA):
Groups of query heads share K, V
Middle ground between MHA and MQA
Used in LLaMA 2, Mistral
4 Sliding Window Attention:
Attend to local window only
Reduces computation for long sequences
Used in Mistral

===== PAGE 23 =====
Position Encoding Methods
Encoding Positional Information:
1 Absolute Sinusoidal (Original Transformer):
Fixed sine/cosine functions
PE(pos,2i) = sin(pos/10000 2i/d )
No learned parameters
2 Learned Absolute (GPT, BERT):
Learnable position embeddings
Limited to training sequence length
3 Relative Position (T5):
Encode relative distances
Better length generalization
4 Rotary Position Embeddings (RoPE):
Encode via rotation matrices
Excellent extrapolation
Used in LLaMA, PaLM
5 ALiBi (Attention with Linear Biases):
Add linear bias to attention scores
Simple, effective extrapolation
