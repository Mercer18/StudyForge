What are Large Language Models (LLMs)?
Definition
Large Language Models are neural networks with billions to trillions of
parameters, trained on massive text corpora, capable of understanding and
generating human-like text with remarkable fluency and knowledge.
Defining Characteristics:
Scale:10B+ parameters (often 100B+)
Data:Trained on trillions of tokens
Compute:Requires massive computational resources
Emergence:Exhibits capabilities not present in smaller models
Generality:Performs well across diverse tasks without fine-tuning
Paradigm Shift:From task-specific models to general-purpose foundation
models

The Scaling Hypothesis
Core Principle:Model performance improves predictably with scale (data,
parameters, compute)
Scaling Laws (Kaplan et al., 2020):
L(N)≈
 Nc
N
αN
Where:
L(N) = Loss with N parameters
Nc = Critical parameter count
αN = Scaling exponent (≈0.076)
Key Insights:
Performance scales as power law with parameters, data, compute
Larger models are more sample-efficient
Emergent abilities appear at certain scale thresholds
Returns diminish but don’t plateau

Architecture of Modern LLMs
Transformer-Based Architecture:
Input Layer:
Token embeddings (vocabulary size×embedding dim)
Positional encodings
Transformer Blocks (N layers):
Multi-head self-attention
Feed-forward networks
Layer normalization
Residual connections
Output Layer:
Linear projection to vocabulary
Softmax for probability distribution
Typical Hyperparameters (GPT-3 175B):
Layers: 96
Hidden size: 12,288
Attention heads: 96
Context window: 2,048 tokens

Training Large Language Models
Training Process:
1 Data Collection:
Web crawl (Common Crawl)
Books, academic papers, code
Filtering and deduplication
Typical size: 300B - 2T+ tokens
2 Pre-training Objective:
Causal language modeling (next token prediction)
Self-supervised learning
Minimizes cross-entropy loss
3 Computational Requirements:
Thousands of GPUs/TPUs
Months of training time
Millions of dollars in compute costs
Advanced distributed training techniques

Distributed Training Techniques
Parallelism Strategies:
1 Data Parallelism:
Replicate model across devices
Split data batches
Synchronize gradients
2 Model Parallelism:
Split model across devices
Each device holds part of model
Required when model doesn’t fit on single device
3 Pipeline Parallelism:
Split model into sequential stages
Process micro-batches in pipeline
Reduces memory, improves utilization
4 Tensor Parallelism:
Split individual layers/tensors
Parallel computation within layer
Used in Megatron-LM

===== PAGE 9 =====
Emergent Abilities of LLMs
Emergence
Abilities that appear suddenly at certain scales, not present in smaller
models, and not explicitly trained for.
Key Emergent Abilities:
1 Few-Shot Learning:
Learn new tasks from few examples
In-context learning without parameter updates
Emerges around 10B+ parameters
2 Chain-of-Thought Reasoning:
Step-by-step problem solving
Improves complex reasoning tasks
Emerges around 100B+ parameters
3 Instruction Following:
Follow complex natural language instructions
Zero-shot task generalization
Enhanced by instruction tuning

===== PAGE 10 =====
In-Context Learning
Concept:LLMs learn to perform new tasks from examples provided in the
prompt, without parameter updates
Learning Paradigms:
Zero-Shot:
Task description only, no examples
”Translate this to French: Hello”
One-Shot:
One example provided
Example + new input
Few-Shot:
Multiple examples (typically 3-10)
Pattern recognition from examples
Advantage:No need for task-specific training data or fine-tuning

Limitations of LLMs
Key Limitations:
1 Hallucinations:
Generate plausible but false information
Confident incorrect responses
Difficult to detect automatically
2 Knowledge Cutoff:
No information beyond training date
Cannot access real-time data
Outdated information on evolving topics
3 Reasoning Limitations:
Struggles with complex multi-step reasoning
Can make logical errors
Limited mathematical precision
4 Context Window:
Limited input length (2K-200K tokens)
Cannot process very long documents end-to-end
Context degradation over long sequences

Hybrid Approaches
Combining SLMs and LLMs:
1 Cascading Models:
SLM for initial filtering/routing
LLM for complex queries only
Cost optimization
2 Distillation Pipeline:
LLM generates training data
SLM learns from LLM outputs
Deploy efficient SLM
3 Specialized + General:
SLM for domain-specific tasks
LLM for general capabilities
Best of both worlds
4 Retrieval-Augmented:
SLM with retrieval mechanism
Access to external knowledge
Bridges capability gap

Summary
Key Takeaways:
1 LLMs are 10B+ parameter models with emergent capabilities
2 Scaling laws show predictable improvements with size
3 LLMs exhibit few-shot learning, reasoning, broad knowledge
4 SLMs offer efficiency, deployability, privacy
5 LLMs provide superior capability, generality, performance
6 Trade-offs: capability vs efficiency, cost vs performance
7 Hybrid approaches can combine strengths of both
8 Model selection depends on use case requirements
Strategic Insight
No one-size-fits-all solution. Choose based on specific requirements:
deployment constraints, task complexity, budget, and performance needs.
